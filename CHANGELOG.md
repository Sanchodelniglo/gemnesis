### Added
- `gemnesis new NAME` ‚Äî scaffolds a hello-world ROM project from a built-in
  template (Ruby config + SGDK C source + 16x16 hero sprite + lab-scene
  background + ROM header generator). Interactive prompts (title, author,
  region) with `--yes`/`-y` to skip and `--here`/`-H` to scaffold into CWD
  instead of `$GEMNESIS_HOME`.
- `gemnesis doctor` ‚Äî environment check: Ruby ‚â• 3.4, Docker on PATH and
  reachable, SGDK image pulled, emulator detected (BlastEm or RetroArch),
  projects home. ‚úì/‚úó/‚öÝ table output.
- `gemnesis build` ‚Äî parses `gemnesis.rb`, writes `src/config.h` +
  `src/boot/rom_head.c`, invokes SGDK image via Docker
  (`docker run --rm --platform linux/amd64 --user $UID:$GID -v $PWD:/src ‚Ä¶`),
  reports `out/rom.bin` size. Auto-pulls image on first run.
- `gemnesis clean` ‚Äî removes `out/`, generated `src/config.h` and
  `src/boot/rom_head.c`, generated resource files, and `.o/.elf/.lst/.map`
  artifacts. Idempotent.
- `gemnesis projects` ‚Äî lists projects in `$GEMNESIS_HOME` with build
  status (time since last build) and ROM size.
- `gemnesis version` (also `--version` / `-V`) ‚Äî prints gem version.
- `$GEMNESIS_HOME` projects directory ‚Äî `gemnesis new` defaults to
  `~/gemnesis/NAME` (or `$GEMNESIS_HOME/NAME`), auto-created.
- Pixel-art lab background (320√ó224 indexed PNG): brick wall, swirling
  portal, CRT monitor with scanlines, shelf of shaded flasks, wood-grain
  counter, tiled floor. Procedurally generated by
  `script/regen_lab_bg_png.rb` (pure Ruby + Zlib).
- ROM header populated from `gemnesis.rb`: title (48-char domestic +
  international), author (16-char copyright), region (U/E/JUE). Written
  to `src/boot/rom_head.c` and protected by a `gemnesis:auto-generated`
  marker ‚Äî strip the marker to take ownership of the file.
- Ruby config DSL (`Gemnesis.configure { |g| ... }`) with validation for
  title (‚â§48 ASCII chars), region (`:ntsc`/`:pal`/`:both`), author.
- `GEMNESIS_SGDK_IMAGE` env var to override the pinned SGDK image.
- Apple Silicon arch detection ‚Äî warns on Rosetta fallback.
- All shell-outs use `Open3` array form ‚Äî no injection via env vars or paths.
- GitHub Actions CI: lint + unit specs on every push; real-Docker
  integration build gated on main pushes via `GEMNESIS_INTEGRATION=1`.
- `.github/` issue + PR templates.

### Changed
- Dropped the planned `gemnesis run` command ‚Äî the gem's contract is to
  produce a playable ROM, not pick the user's emulator. Run the resulting
  `out/rom.bin` in RetroArch, BlastEm, or any Mega Drive emulator of choice.
- SGDK image pinned by digest (was `:latest`) ‚Äî upgrade requires an
  explicit pull + digest bump, preventing surprise breakage from upstream.
- `--user $UID:$GID` passed to `docker run` so files written to `/src`
  match the host user (fixes CI permission errors on Linux runners).

### Security
- `gemnesis.rb` is loaded via `Kernel.load` ‚Äî the trust model matches a
  `Makefile`/`Rakefile`: it runs Ruby in the user's own project, treated
  as user-owned code. Documented in source comments.
